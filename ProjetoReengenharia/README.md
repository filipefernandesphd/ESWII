# Instruções

### Objetivo geral

Aplicar os princípios de reengenharia de software para melhorar a qualidade, modularidade e usabilidade do sistema, mantendo sua lógica de negócio original.

### Objetivos específicos

- Compreender o funcionamento do código legado (engenharia reversa)
- Analisar e documentar a estrutura atual (diagramas e descrições)
- Propor melhorias estruturais no código (refatoração, remoção de code smells, padrões de projetos etc.)
- Desenvolver uma interface web simples
- Implementação de testes unitários

### Dinâmica

- Grupos de 2 a 3 alunos
- Duração de 4 aulas
- Cada grupo terá 7 min para apresentar o sistema reestruturado, considerando os tópicos abaixo:
    - Procedimentos adotados pela equipe para compreensão do sistema
    - Organização da equipe para reestruturar o sistema (processos, papéis e responsabilidades)
    - Decisões estratégicas de engenharia de software para reestruturar o sistema (técnicas, métodos e ferramentas)
    - Mecanismos para documentação do sistema
    - Regras para garantir a qualidade na evolução e manutenção futura do sistema
    - Mostrar o sistema reestruturado funcionando, evidenciando as principais diferenças do sistema legado
    - A melhor equipe ganhará um prêmio
